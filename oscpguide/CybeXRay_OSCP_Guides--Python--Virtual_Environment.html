Virtual Environment


Python Virtual Environments




Python virtual environments are used to keep separate versions of same package/modules in different virtualized environments.

Preparation



1. Create a directory to store your virtual environment.
Note: The virtual environments are created & stored in the form of directories inside the current running directory.

Install Any One
2. Install venv
The above module should come as default in python3. But, if its not present we can install by the following.

apt install python3.10-venv

Note: venv is newer and comes in default in some python installations. However, the python version of the environment will be the same as the python version running the venv module.

3. Install virtualenv
The above module is older, but it has one advantage over venv.
virtualenv can be used to specify a python version in our virtual environment.

apt install python3-virtualenv
Or
pip install virtualenv [Used for version specific installation eg. pip2, pip3 ]


Note: To view packages installed in system or a virtual environment.
pip list
or
pip list --local [To see packages specific to an environment]

Creating the Environment



Using venv:

python -m venv <virtual_env_name>

Eg. python -m venv my_env
Note: It creates the environment in the same directory. However, we can also use absolute path while giving the name.

Using virtualenv:

virtualenv <virtual_env_name>

Eg. virtualenv my_env
Note: It creates the environment in the same directory. However, we can also use absolute path while giving the name.

To create a virtual environment with specific python version.

virtualenv -p <python_version> <virtual_env_name>

Eg. virtualenv -p python3.9 my_env

Activating the Environment



Linux/Mac:

source <Path_to_Virtual_Environment_directory>/bin/activate

eg. source test_env7/bin/activate
Note: Once successfully done. Our prompt will have the virtual environment name.

Windows(Powershell):

PS> <Path_to_Virtual_Environment_directory>/scripts/activate

eg. PS> test_env7/scripts/activate

Deactivate the Environment



deactivate

Deleting the Environment



Inside the Virtual Environments directory.

rm -rf <virtual_env_name>

Creating a Virtual Environment with all system modules/packages



Note: Can be done with both venv & virtualenv

While creating the environment add “--system-site-packages”

python -m venv <virtual_env_name> --system-site-packages
or
virtualenv <virtual_env_name> --system-site-packages

To check all the Modules in the environment. Once we activate it Run.
For all Packages:
pip list

For Environment specific packages:
pip list --local

Creating a Requirements.txt File



Once inside an activated virtual environment, we can use:

pip freeze > requirements.txt [For All Modules/Packages]
Or
pip freeze --local > requirements.txt [For Environment Specific Modules/Packages]

To Install Modules/Packages from requirements.txt

pip install -r requirements.txt
